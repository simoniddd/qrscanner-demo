# QR Scanner Demo

Демо-приложение для сканирования QR-кодов на Android с использованием CameraX и ML Kit. Включает оптимизации для надёжного распознавания: автозум для мелких кодов, ROI-обработку, троттлинг FPS и компенсацию экспозиции.

## Возможности

- **Сканирование QR-кодов в реальном времени** — CameraX + ML Kit Barcode Scanning
- **Автоматический зум** — при обнаружении мелкого QR-кода камера плавно приближается (ZoomSuggestionOptions API)
- **ROI (Region of Interest)** — анализируется только центральные 50% кадра, что снижает нагрузку на CPU
- **Троттлинг FPS** — ограничение до 20 кадров/с (интервал 50 мс), увеличивает время экспозиции сенсора и уменьшает motion blur
- **Компенсация экспозиции** — индекс -2 для корректного считывания с ярких экранов (мониторы, телефоны)
- **Оптимизация разрешения** — формат 4:3 (нативный для сенсора), целевое разрешение 1080p без программного кропа
- **Проверка стабильности** — QR-код подтверждается только после стабилизации позиции (допуск 20 px, задержка 300 мс)
- **Анимированная рамка сканирования** — статическая рамка 244×244 dp с плавной анимацией при обнаружении кода

## Оптимизации сканирования

### ROI (Region of Interest)

Вместо анализа всего кадра обрабатывается только центральная область (50% ширины/высоты). Вокруг ROI добавляется отступ 25% для "тихой зоны" QR-кода. Это значительно снижает нагрузку на ML Kit.

### Автоматический зум

При обнаружении QR-кода, который не удаётся прочитать из-за малого размера, ML Kit через `ZoomSuggestionOptions` предлагает коэффициент зума. Камера плавно приближается (анимация 300 мс, `DecelerateInterpolator`). Базовый зум — 1.3x. Порог изменения — 0.1x для предотвращения дрожания.

### Троттлинг FPS

Частота анализа кадров ограничена до 20 FPS (интервал 50 мс). При более редком захвате сенсор получает больше времени экспозиции, что снижает размытие при движении руки.

### Компенсация экспозиции

Индекс экспозиции установлен в -2, что помогает считывать QR-коды с ярких дисплеев (мониторы, экраны телефонов), где стандартная экспозиция приводит к засветке.

### Проверка стабильности

Перед подтверждением сканирования проверяется стабильность положения QR-кода:
- Координаты рамки не должны меняться более чем на 20 px
- Стабильность должна сохраняться 300 мс

Это предотвращает ложные срабатывания при движении камеры.

## Ключевые константы

| Константа | Значение | Описание |
|-----------|----------|----------|
| `DEFAULT_ZOOM_RATIO` | 1.3 | Базовый коэффициент зума |
| `ZOOM_CHANGE_THRESHOLD` | 0.1 | Минимальное изменение зума |
| `ZOOM_ANIMATION_DURATION_MS` | 300 мс | Длительность анимации зума |
| `ANALYSIS_INTERVAL_MS` | 50 мс | Интервал анализа кадров (20 FPS) |
| `ROI_SIZE_RATIO` | 0.5 | Размер ROI относительно кадра |
| `EXPOSURE_COMPENSATION_INDEX` | -2 | Компенсация экспозиции |
| `BOUNDS_TOLERANCE` | 20 px | Допуск стабильности координат |
| `QR_STABILITY_DELAY_MS` | 300 мс | Задержка подтверждения сканирования |
| `ANIMATION_DELAY_MS` | 300 мс | Задержка перед показом результата |

## Сборка и запуск

```bash
# Клонирование
git clone <repository-url>
cd qrscanner-demo

# Сборка
./gradlew assembleDebug

# Установка на устройство
./gradlew installDebug
```

Или откройте проект в Android Studio (Ladybug или новее) и запустите на устройстве/эмуляторе.

## Пользовательский сценарий

1. При первом запуске приложение запрашивает разрешение на камеру
2. После предоставления разрешения открывается экран сканирования с камерой и рамкой
3. Наведите камеру на QR-код — при обнаружении рамка анимируется к позиции кода
4. Если код мелкий — камера автоматически приблизится
5. После стабильного распознавания отображается экран с результатом
6. Кнопка «Scan Again» возвращает к сканированию
